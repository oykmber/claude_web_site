# CLAUDE.md

## コミュニケーションルール

要件が不明確な場合や判断材料が不足している場合は、実装前にユーザーへ質問すること。

特に以下の場面では必ず確認する：
- 技術選定
- UI/UXの方向性
- 機能の優先順位
- トレードオフが発生する判断

## 技術情報のルール

【必須】技術的な実装を行う前に、必ずWebSearchで公式ドキュメントの最新情報を確認する。
確認なしでの実装は禁止。

対象：
- ライブラリ・フレームワークの導入
- API・設定方法の実装
- ベストプラクティスの適用

## コードレビューのルール

【必須】コード改修が完了したら、必ず `code-reviewer` サブエージェントを使用してレビューを実行すること。

対象：
- 新機能の実装完了時
- バグ修正の完了時
- リファクタリングの完了時

手順：
1. 改修作業を完了する
2. `code-reviewer` サブエージェントにレビューを依頼する
3. レビュー結果に基づき対応する：
   - 🔴 Critical → 必ず修正してから再レビューを実施
   - 🟡 Warning → 修正を実施（ユーザー確認不要）
   - 🔵 Suggestion → ユーザーに提示し判断を仰ぐ
4. Critical/Warningの修正後は再度レビューを実施し、問題がなくなるまで繰り返す
